# 维度提升与投影量化

## 本次规划
- 解释 `20to8`、`24to8` 这类配置在代码中的含义。
- 梳理投影矩阵 `M` 的训练、最近邻搜索和最终权重表示。
- 只基于当前代码可验证的机制，不引入论文外推细节。

## `expc` 的含义
`expc` 字符串形如 `24to8`，在 `TmpLinear` 和 `FWTLinear` 中被拆成：
- `root2 = 24`：提升后的码字维度。
- `root = 8`：投影后的目标向量维度。

直观理解：每个 8 维归一化权重块，不是直接在 8 维空间里做低比特均匀量化，而是寻找一个更高维二值码字，再通过投影矩阵映射回 8 维空间。这样可以用二值码字组合出更密的投影点集。

README 中给出的经验配置是：
- 3-bit 模型使用 `24to8`。
- 2.5-bit 模型使用 `20to8`。

## 投影矩阵 `M`
`FWTLinear.bit_channel_convert()` 会从 `./lattice/{expc}.pt` 读取矩阵 `T`。该矩阵把高维二值码字投影到低维目标空间。

如果需要重新生成矩阵，入口是 `LiftQuant/lattice_generator2.py`。该脚本通过随机初始化矩阵 `M`，重复采样目标向量 `z_batch`，寻找二值候选码字对应的投影点，并优化投影点到目标向量的距离。

当前脚本默认写死为：
- `D_in = 32`
- `D_out = 16`
- 保存路径 `./lattice/32to16.pt`

因此若要生成 `20to8` 或 `24to8`，需要有意识地修改脚本维度，而不是直接运行默认脚本期待得到 README 配置。

## 最近邻搜索
`FWTLinear.find_nearest_fast()` 是主线快速搜索逻辑。它先对投影矩阵做 SVD，构造零空间基 `N`，把 `M` 扩展成可逆方阵 `M_square`。

对每个低维目标向量 `z`，代码枚举缺失维度上的 `[-1, 1]` padding，拼成候选高维向量，再乘以 `M_square_inv` 并取符号得到二值码字。随后把候选码字投影回低维空间，选择距离 `z` 最近的码字。

这相当于把不可枚举的完整高维二值空间，缩小到与当前目标向量相关的一组候选，从而让 `20to8`、`24to8` 这类配置可实际运行。

## 从临时权重到最终权重
`TmpLinear` 在训练阶段保留浮点原始权重和可学习参数，通过 `quant_tmpweight()` 临时构造量化权重参与前向。

`FWTLinear.convert_form_tmplinear()` 会把训练后的状态转换为部署形式：
- 保存 `scale`、`a1`、`a2`。
- 将权重变换到投影域。
- 根据 `training_trans` 选择可训练变换或 Hadamard 变换。

`FWTLinear.bit_channel_convert()` 进一步把每个 `root` 维块映射为 `root2` 维二值码字，并修改右侧变换矩阵，使最终前向仍能还原到原始线性层需要的权重空间。

## 打包存储
`FWTLinear.pack_to_int8()` 把量化后的低比特值按 8 个一组打包为 `uint8`。注意这里的 `uint8` 是存储容器，不表示权重是 8-bit 量化；真实有效位宽由 `wbits` 和 `expc` 共同决定。
